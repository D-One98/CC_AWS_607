# AWS Service Setup Guide: S3 (Simple Storage Service)

## 1. Overview
S3 is AWS's object storage service — files ("objects") organized into globally-unique "buckets". It is used for backups, static assets, data lakes, and hosting static websites. During the internship S3 was used to store screenshots, upload sample files via console and CLI, set object-level permissions, and later serve as the origin for a static website (see the Static Website Hosting doc).

## 2. Step-by-Step Setup Guide
1. Open the **S3** console → **Create bucket**.
2. Bucket name: `vineet-shukla-internship-2026` (must be globally unique, lowercase, no underscores).
3. Region: `ap-south-1` (Mumbai).
4. **Block Public Access:** keep all four boxes **ON** for private storage (turn them off only for the static-website bucket).
5. Bucket versioning: **Enable** (so overwrites and deletes are recoverable).
6. Default encryption: **SSE-S3** (Amazon-managed keys). Create bucket.
7. Open the bucket → **Upload → Add files** → pick a test file (e.g., `hello.txt`) → **Upload**.
8. Open the uploaded object → **Object URL** — accessing it in a browser should return **AccessDenied** (proving the bucket is private).
9. (Optional CLI test) After configuring `aws configure` with an IAM access key, run:
   ```
   aws s3 ls s3://vineet-shukla-internship-2026
   aws s3 cp local.txt s3://vineet-shukla-internship-2026/
   ```

## 3. Key Configurations Used
* **Bucket name:** `vineet-shukla-internship-2026`
* **Region:** `ap-south-1`
* **Block Public Access:** ON (all 4 settings)
* **Versioning:** Enabled
* **Encryption:** SSE-S3 (AES-256)
* **Storage class of uploads:** Standard
