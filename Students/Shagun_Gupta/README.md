# AWS Cloud Engineering

## What is AWS?

Amazon Web Services (AWS) is a cloud computing platform provided by Amazon. It offers a wide range of on-demand cloud services such as computing power, storage, databases, networking, security, analytics, and machine learning. Instead of purchasing and maintaining physical servers, organizations can use AWS resources over the internet and pay only for what they use.

AWS enables individuals, startups, enterprises, and governments to build, deploy, and scale applications quickly while reducing infrastructure costs.

## Why Use AWS?

AWS helps organizations:

* Eliminate the need to purchase and maintain physical servers.
* Scale infrastructure up or down based on demand.
* Improve application availability and reliability.
* Reduce operational and maintenance costs.
* Deploy applications globally with minimal effort.
* Access a large ecosystem of managed cloud services.

## Key Characteristics of AWS

* **On-Demand:** Resources can be created whenever required.
* **Pay-as-You-Go:** You pay only for the resources you consume.
* **Scalable:** Resources can be increased or decreased based on workload.
* **Highly Available:** Services are designed for reliability using multiple Availability Zones and Regions.
* **Secure:** AWS provides identity management, encryption, monitoring, and compliance features.
* **Global Infrastructure:** AWS operates data centers across multiple Regions worldwide.

## Advantages of Cloud Computing

* Lower infrastructure costs
* Faster deployment
* High availability
* Automatic scaling
* Disaster recovery capabilities
* Global accessibility
* Managed services that reduce operational overhead

## AWS Global Infrastructure

AWS delivers services through a global infrastructure consisting of:

* **Regions** – Separate geographic areas where AWS operates data centers.
* **Availability Zones (AZs)** – One or more isolated data centers within a Region that provide fault tolerance and high availability.
* **Edge Locations** – Locations used primarily by Amazon CloudFront and other services to deliver content with low latency.

## Core Categories of AWS Services

AWS offers hundreds of cloud services that can be grouped into several core categories:

| Category       | Purpose                                            | AWS Service Example |
| -------------- | -------------------------------------------------- | ------------------- |
| **Compute**    | Runs applications and code.                        | Amazon EC2          |
| **Storage**    | Stores files, backups, and other data.             | Amazon S3 (Buckets) |
| **Networking** | Connects and secures cloud resources.              | Amazon VPC          |
| **Databases**  | Stores structured data with querying capabilities. | Amazon RDS          |

These categories form the foundation of most cloud architectures. Throughout this repository, you'll gain hands-on experience with each of these core AWS services.
